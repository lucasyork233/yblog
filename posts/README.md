# 文章目录说明

这个目录存放所有的博客文章，每个文章都是一个独立的 JavaScript 文件。

## 如何添加新文章

### 1. 创建新文件

- 在 `posts/` 目录下创建新文件，命名格式：
    - 简单文章：`post-编号.js`（例如：`post-003.js`）
    - 复杂标题：`自定义名称.js`（例如：`markdown-guide.js`）

- **注意**：文件名中的连字符会自动转换为驼峰命名作为变量名：
    - `post-003.js` → `window.post003`
    - `markdown-guide.js` → `window.markdownGuide`
    - `test-markdown.js` → `window.testMarkdown`

### 2. 编写文章内容

按照以下模板编写：

```javascript
// 文章：文章标题
window.post003 = {
  slug: 'post-003',          // 唯一标识符，必须与文件名对应
  title: '文章标题',          // 文章标题
  date: '2026-01-06',        // 发布日期
  summary: '文章简短描述',    // 简短摘要
  tags: ['tag1', 'tag2'],    // 标签数组
  content: `文章内容
## 章节标题

这里是文章的详细内容。

- 支持列表
- 支持 **粗体**
- 支持 *斜体*

\`\`\`javascript
// 支持代码块
function hello() {
  console.log('Hello World');
}
\`\`\`

> 支持引用
`
};
```

- **注意**：
    - 必须使用 `window.post003 = {}` 的形式，而不是 `export`
    - 变量名必须与文件名转换后的驼峰命名一致
    - 浏览器直接加载脚本文件，所以需要挂载到 `window` 对象上

### 3. 更新配置文件

在 `posts/config.js` 中添加新文件名：

```javascript
const POST_FILES = [
  'post-001.js',
  'post-002.js',
  'post-003.js',  // 添加新文章
];
```

### 4. 刷新浏览器

保存所有文件后，刷新浏览器即可看到新文章。

## 支持的 Markdown 语法

- **标题**：`#`, `##`, `###`
- **粗体**：`**文本**`
- **斜体**：`*文本*`
- **代码块**：``` ```javascript ... ``` ```
- **行内代码**：`` `code` ``
- **无序列表**：`- 项目`
- **有序列表**：`1. 项目`
- **引用**：`> 引用文本`

## 📋 **完整添加流程**

1. **创建文章文件**：`posts/post-003.js`
2. **编写内容**：使用模板
3. **更新配置**：在 `posts/config.js` 中添加 `'post-003.js'`
4. **刷新浏览器**：完成！

> 💡 **提示**：虽然需要手动更新配置文件，但这是最可靠的方式，避免了复杂的目录扫描逻辑。
